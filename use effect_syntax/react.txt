1.Why React Was Launched ‚Äî The Main Reasons
Too much manual DOM manipulation
Before React, developers had to use plain JavaScript or jQuery to update HTML elements manually.

React introduced the Virtual DOM, which updates only what‚Äôs necessary ‚Äî making apps much faster.

4.what is typescritpt
Definition
TypeScript is a superset of JavaScript ‚Äî meaning it builds on top of JavaScript by adding types.
So every valid JavaScript code is also valid TypeScript code, but TypeScript gives you extra features like type checking and better code tools.

Static Typing:
You can define data types (like string, number, boolean) to catch errors before running the code.

Compile-time Checking:
TypeScript code is checked before running, so you can find bugs early.

Object-Oriented Features:
It supports classes, interfaces, and inheritance, making code more structured

Better Tooling:
Editors like VS Code give smart suggestions, autocompletion, and error highlighting.


what is variable
A variable is a name used to store data in a program ‚Äî like a box that holds a value.
You can put data in it, use it, and change it later.
example
let name = "Ranjith";
let age = 21;

name and age are variables.
"Ranjith" (text) and 21 (number) are the values stored in them.

Why We Use Variables
To store data (like name, number, or result)
To reuse values easily
To change data while the program runs

| Keyword | Meaning                           |
| ------- | --------------------------------- |
| `var`   | Old way (not recommended now)     |
| `let`   | Modern, allows value changes      |
| `const` | Used for values that never change |

7.what is black scope
Block scope means a variable is only available (can be used) inside the block { } where it is declared.
A block is any code written inside { }, like in if statements, loops, or functions.
example
{
  let name = "Ranjith";
  console.log(name); // ‚úÖ Works here
}

console.log(name);   //  Error: name is not defined


8.what is function scope
Function scope means a variable is only available inside the function where it is declared.
You cannot use it outside that function.
variables declared inside a function belong only to that function
example
function greet() {
  let name = "Ranjith";  // function-scoped variable
  console.log("Hello, " + name); //  Works here
}

greet();       // Output: Hello, Ranjith
console.log(name); // Error: name is not defined


9.what is hosting scope?
Hoisting is JavaScript‚Äôs behavior of moving variable and function declarations to the top of their scope before code execution.
Only the declaration is hoisted, not the initialization.
This applies to var variables and function declarations.

11. what is lixical scope?
Lexical scope means:
The scope of a variable is determined by where it is written in the code (the structure of the code), not by where it is called.
A function can access variables that were defined in the place where the function was created.

 For beginners or in a project:
